# ServicePro CRM

Фронтенд CRM-система для сервісного центру. Дані зберігаються в `localStorage`, бекенд не потрібен.

## Технології

- HTML, CSS, JavaScript (ES модулі)
- Tailwind CSS, Chart.js, Font Awesome
- LocalStorage для збереження даних

## Структура проекту

```
js/
├── app.js              — точка входу
└── modules/
    ├── auth.js         — автентифікація
    ├── database.js     — робота з даними
    ├── router.js       — навігація
    ├── ui.js           — модалки, тости
    ├── dashboard.js    — головна сторінка
    ├── orders.js       — замовлення
    ├── clients.js      — клієнти
    ├── services.js     — послуги
    ├── inventory.js    — склад (запчастини)
    ├── finance.js      — фінанси, каса
    ├── settings.js     — налаштування (загальні)
    ├── printEditor.js  — редактор полів для друку
    └── roleSettings.js — налаштування ролей
```

## Основні можливості

- **Дашборд** — статистика, графіки
- **Замовлення** — список, форма, Kanban-дошка, друк актів
- **Клієнти** — база клієнтів, історія замовлень
- **Склад** — запчастини, артикули
- **Послуги** — прайс послуг
- **Фінанси** — каса, транзакції
- **Налаштування** — ролі, користувачі, друк

## Реалізовані оновлення

### Автентифікація

- Вхід через вибір користувача з бази
- Паролі для користувачів (якщо є хоча б один адмін)
- Якщо немає адмінів — вхід за ролями без пароля

### Ролі та користувачі

- **Налаштування ролей** — окрема сторінка (`/roleSettings`), доступна з Налаштувань
  - Права доступу для Адміністратора, Менеджера, Майстра
  - Пункти меню для кожної ролі
- **Користувачі** — додавання майстрів та адміністраторів
- Встановлення пароля для кожного користувача

### Печатні документи

- **Редактор полів для друку** — окрема сторінка (`/printEditor`), доступна з Налаштувань
  - Формат паперу: А4 або 58 мм (термопринтер)
  - Назва компанії (виконавець)
  - Адреса та телефон компанії
  - Заголовок документа (наприклад, «АКТ ВИКОНАНИХ РОБІТ»)
- Налаштування зберігаються в `Database.data.printConfig`

### Клієнти

- Редагування клієнтів
- Декілька телефонів при створенні замовлення та в картці клієнта

### Інвентар

- Автоматична генерація артикулу (PART-0001), якщо поле порожнє
- Можливість видаляти запчастини

### Мобільна версія

- Адаптивний інтерфейс, safe-area
- Кнопка «Нове замовлення» у списку замовлень
- Kanban зі scroll-snap
- Toast-повідомлення з урахуванням safe-area

### Інше

- Виправлення в router, clients, orders, inventory, database
- Експорт БД в JSON

## Як запустити

1. Відкрити `index.html` у браузері
2. Або використовувати локальний сервер (наприклад, `npx serve .`)
